グラフのパラメータを変数渡しで決定する実験、
及び、static周りの環境整備についてまとめる。

グラフのパラメータについて
formタグを使うと、変数を渡せる。以上・・？
まず、手入力用にhtmlを追加で用意する。
urlはmake_plot/とする（変数は渡さない）。
で、formタグで4つのインプットボックスを囲む。
インプットボックスにはそれぞれind_3, ind_2, ind_1, ind_0の名前を付けておく。
イメージ的にはこれらがa, b, c, dで、グラフはy=ax^3+bx^2+cx+dにする予定。
アクションに同じページのアドレスを設定することで、グラフがロードされると同時に同じページに戻るようにする。
グラフのアドレスをurlでview関数に紐付ける。
ここのアドレスをplot_2/<ind_3>/<ind_2>/<ind_1>/<ind_0>/とする（型はあえて決めない）。
get_svg_customでこれら4つの引数を受け取り、グラフを描画する。
ここの変数をどうやって決めるかというと、
このページ（make_plot/）のview関数でrequest.POST['']を使ってフォームから送信された値を受け取っている。
エラー処理が若干適当ですが・・デフォルトで0, 0, 0, 1としている（じゃないと最初のアクセスで・・）。
インプットボックスでクリック時に中身がリセットされるとかそういうのは後でいいので。
get_svg_customでその引数を受け取り、・・intにしてる。doubleの方がいいんだろうけど。
というか文字列に対してエラーを出したいんよね（ValueErrorが送出されます）。
0/0/0/1/

static周りの環境について。
アプリフォルダuse_plt直下にstaticを用意して、その中にまたuse_pltを作り、
その中にimagesフォルダとあとstyle.cssファイルを用意する。
base.htmlをいじる。いつもの<link rel = "stylesheet" type = "text/css" href = "ここ">
で、その前に
{% load static %}
する。タグの前。で、href = "{% static '～～' %}" って書くと、～～がstatic直下のアドレスになる。
だから、use_plt/style.cssって書けばstyle.cssを呼び出せるし、
use_plt/images/Niko_rue.pngって書けば画像が出てくる。
